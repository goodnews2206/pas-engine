# PAS — Memory Architecture

Memory is the heart of PAS. The product's differentiator is not that it sends a fast text — it's that it *remembers*. PAS organizes memory into a four-tier hierarchy, from the most ephemeral to the most durable.

## The hierarchy

```
Session Memory  →  User Memory  →  Operational Memory  →  PAS Brain
   (ephemeral)      (per person)      (per brokerage)       (durable, institutional)
```

Each tier feeds the next. Lower tiers are transient and contextual; higher tiers are durable and institutional.

### 1. Session Memory
- **Scope:** a single interaction / session.
- **Lifespan:** ephemeral — relevant to the current exchange.
- **Purpose:** hold the immediate context needed to act right now (the active lead, the current conversation, the in-flight task).
- **Analogy:** working memory / RAM.

### 2. User Memory
- **Scope:** an individual person (an agent, ISA, team lead).
- **Lifespan:** persists across that user's sessions.
- **Purpose:** remember what *this person* knows, does, and is responsible for — their leads, their patterns, their history.
- **Analogy:** a personal assistant who remembers your week.

### 3. Operational Memory
- **Scope:** the brokerage's operations as a whole.
- **Lifespan:** persistent across users and time.
- **Purpose:** the shared operational state — leads in flight, follow-up status, who-did-what, accountability and operational visibility across the team. This is where "did the follow-up actually happen?" is answered.
- **Analogy:** the operations control room.

### 4. PAS Brain
- **Scope:** the entire brokerage as an institution.
- **Lifespan:** durable — the long-term, accumulating memory.
- **Purpose:** **brokerage memory** that survives staff turnover, handoffs, and time. It's the institutional record: every lead, every pattern, every interaction, retained. When an ISA quits or an agent leaves, the knowledge doesn't walk out the door — it stays in the PAS Brain.
- **Analogy:** the brokerage's permanent institutional memory and intelligence.

## Why the hierarchy matters

- **Continuity without humans:** because state flows up into Operational Memory and the PAS Brain, follow-up and accountability don't depend on whether a person remembered.
- **Turnover resilience:** the single biggest hidden cost in brokerages is that knowledge lives in people's heads and CRMs they don't maintain. The PAS Brain is the antidote — it's the reason "brokerage memory" is a core content topic.
- **Operational visibility:** Operational Memory is what makes it possible to *see* whether things actually happened, not just that they were assigned.

## How to talk about it (for content + product)

- Lead with **memory and continuity**, not features.
- The PAS Brain is the headline idea: *"Your brokerage finally has a memory."*
- Tie each tier to a real pain: Session → speed, User → personal follow-through, Operational → visibility/accountability, PAS Brain → turnover resilience and institutional knowledge.

## What it is NOT

- Not a CRM's contact list (that's a static database; this is layered, living memory).
- Not chat history (Session Memory is the smallest tier, not the product).
- Not analytics dashboards bolted on after the fact — memory is the substrate, not a report.
